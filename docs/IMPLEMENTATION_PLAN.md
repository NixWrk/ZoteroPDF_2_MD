## Реализация BUG-A и BUG-B (commit 6af0ddb)

### BUG-A: Дублирование номеров в списке литературы

**Решение:** Добавлен идемпотентный guard в `ensure_visible_ref_number()` функцию (`single_file_html.py`).
- Перед добавлением `<span class="z2m-ref-num">N.</span>` проверяем наличие этого класса в body
- Если уже есть — пропускаем обработку (уже обработано в предыдущем polish pass)
- Код: 4 строки, минимальный риск

**Файл:** `src/zoteropdf2md/single_file_html.py`, функция `ensure_visible_ref_number` (строка ~837)

### BUG-B: Несогласованный перевод заголовка Wang

**Решение:** Реализована merge/split логика для текстовых узлов в заголовках:

1. **Пред-merge** (`_merge_heading_text_nodes`): перед переводом объединяются текстовые узлы внутри `<h1>-<h6>` через separator `\x00`
   - Пример: `["A Novel Passive", "LC", "Sensor"]` → `"A Novel Passive\x00LC\x00Sensor"` (один сегмент)
   - Это предоставляет контекст модели: "Sensor" теперь видит предыдущее слово

2. **Пост-split** (`_split_heading_text_nodes`): после перевода восстанавливаются исходные узлы
   - Если модель сохранила separator `\x00` в переводе — разбиваем результат обратно
   - Если модель потеряла separator — оставляем merged результат в первом узле

3. **Fallback** в `_fix_heading_translation_breaks()`: дополнительная защита
   - Понижаем регистр первого кириллического слова после `</i>`, `</em>`, `</b>`, `</strong>`
   - Не исправляет падеж, но убирает заглавную букву, когда merge не сработал

**Файлы:**
- `src/zoteropdf2md/translategemma.py`: `_merge_heading_text_nodes()`, `_split_heading_text_nodes()`, интеграция в `translate_html_text_nodes()`
- `src/zoteropdf2md/single_file_html.py`: расширена `_fix_heading_translation_breaks()`

**Сложность:** средняя. Merge логика требует осторожности при индексировании parts[].

## Универсальность (сквозные принципы)

1. **Валидация по реальным якорям библиографии.** Обёртки `<sup>N</sup>`/`[N]` → `<a href="#ref-N">` только если `id="ref-N"` существует в документе.
2. **Плейсхолдер-изоляция** непереводимого контента (аббревиатуры, URL, `<code>`, атрибуты).
3. **Конфигурируемые словари** — аббревиатуры, термины-заголовки выносятся из кода в `abbreviations.py`.
4. **Узкие триггеры** для math-распаковки и caption-чистки — не трогать реальные формулы/код.
5. Все фиксы универсальны: проверяются на обоих документах (Wang + Teo), не подгоняются под один PDF.

- «версия 2.0», «таблица 3.4», «GPT-4», «страница 17.68 содержит 5%» — не ссылки.
- Реальные формулы с буквами/операторами — не разворачиваются в sup.
- `<sup>a</sup>`, `<sup>b</sup>` (короткие маркеры таблиц) — не трогаются.

- `DeepSeek-R1`, `GPT-4` не кликабельны как ссылки.
- Отсутствуют рядом `AI` и `ИИ`.
- Заголовок статьи и таблиц — связный перевод.
- Нет фрагментов промпта в теле.
