# Финальный план: дефекты HTML-пайплайна PDF→HTML→RU

## Контекст

Пайплайн `run_headless.py` преобразует PDF из Zotero → Marker HTML → single-file HTML с инлайн-картинками → RU-перевод TranslateGemma.

Проведён перепрогон на двух коллекциях (`md_output/intracranial/`, `md_output/llm_medicine/`). Сравнил результат с тремя источниками дефектов:

1. **Исходная диагностика** (5 дефектов: картинка Figure 2, DeepSeek-R1, аббревиатуры, нераспознанные ссылки, ссылки-формулы).
2. **Новые скриншоты Wang + Teo** (9 дополнительных дефектов).
3. **Файлы анализа в корне репо** (объединены в этот документ).

Этот документ — **единая правда** о состоянии работ и приоритетах.

## Новые дефекты (Wang + Teo, скриншоты)

### Группа A — Артефакты перевода
- **A1. Leak промпта.** В Teo в тело документа попал фрагмент промпта:
  `«LC, VNA, ICP, SNR, ADC, IEEE, GAI, RF, MEMS, FPGA, AC, DC, USB, MIMO - перевод на русский язык.»` Модель скопировала список защищённых аббревиатур в вывод.
- **A2. Прерывистый/слипшийся перевод.** `«для создания решений10Модели рассуждения…»` — `<sup>10</sup>` склеен со следующим словом без пробела и без корректной обёртки.
- **A3. Некорректный перевод заголовка Wang.** `«Новая схема считывания внутричерепного давления для пассивной беспроводной связи LC Датчик»` — неверное членение: «LC» оторвано, «Датчик» с большой буквы. Должно быть `«…беспроводного LC-датчика»`.
- **A4. Таблицы: дубль EN+RU и плохой перевод.** `«Таблица IV СРАВНЕНИЕ СОСТОЯНИЯ ИСКУШЕНИЙ»` (должно быть «…СОВРЕМЕННЫХ РЕШЕНИЙ»), рядом остаётся `«TABLE IV COMPARISON OF STATE OF ARTS»`.

### Группа B — Размещение изображений и навигация
- **B1. Рисунки разрывают предложения.** В Wang: `«…это аналоговый <img Рис.1> с ограниченной частотной разверткой»`. Marker/HTML поставил `<figure>` внутрь `<p>`.
- **B2. Клик по ссылке на рисунок скрывает сам рисунок.** Якорь `id="fig-N"` стоит у `<p>` с подписью → при scroll-to-anchor браузер выравнивает caption по верху, картинка выше — за пределами экрана.

### Группа C — Ссылки и URL
- **C1. Склеенные URL.** В Teo: `«https://www.wysa.com/and https://woebothealth.com/(ref. 30).»` — потерян пробел, `and` прилипло, `(ref. 30)` остался как текст вместо sup-ссылки.
- **C2. Нераспознанные bare-citations.** `«клиническим наборам данных 62.»`, `«проблемы 17.68.»`, `«специализированных знаний 67.»` — цифры после пробела с точкой-разделителем, `_BARE_CITATION_DOT_PATTERN` и spaced-паттерн не срабатывают.

### Группа D — Артефакты caption и OCR-символов
- **D1. Символы `\ | \` и одиночные `\` в подписях.** В Teo: `«Fig. 2 \ | \ GAI development pipeline … \ The \ key steps …»`. Частично уже обрабатывается `_SLASH_PIPE_ARTIFACT_PATTERN`, но одиночные `\` между словами остаются.

## Pipeline изменений: коммиты

### ✅ Commit 1 — Inline images into md_output HTML *(in progress)*
**Что:** Вызвать `inline_images_from_html_file()` на EN- и RU-HTML сразу после их появления в `md_output/`.  
**Где:** `src/zoteropdf2md/pipeline.py` (новый блок перед строкой 520 + после строки 567).  
**Критерий:** `<img[^>]+src="(?!data:)` по `md_output/**/*.html` = 0.

### ✅ Commit 2 — Stabilize Latin abbreviations *(next)*
**Что:** Двухслойная защита аббревиатур.  
**Слой 1 (translategemma.py):** плейсхолдер-маскирование `\b(AI|GAI|LLM|GPT|NLP|RAG|FDA|ML|DL|RLHF|RLAIF|...)\b` перед вызовом модели, восстановление после.  
**Слой 2 (single_file_html.py:939–953):** расширить `_LATIN_ABBREV_RESTORE_MAP` (добавить `ИИ→AI`, `ЛЛМ→LLM`, `НЛП→NLP`, `ГПТ→GPT`, `РАГ→RAG`), учитывать падежные суффиксы `ИИ-моделей`, `ГАИ-систем`, применять вне библиографии и вне `<code>/<a>/<cite>`.  
Общий словарь вынести в `src/zoteropdf2md/abbreviations.py`.  
**Критерий:** `\bИИ\b|\bГАИ\b|\bЛЛМ\b|\bНЛП\b|\bГПТ\b|\bРАГ\b` вне библиографии = 0.

### 🆕 Commit 3 — Prompt-leak guard (A1)
**Что:** Защита от эхо-копии списка аббревиатур из промпта в вывод.  
**Где:** `src/zoteropdf2md/translategemma.py`.  
**Как:**
1. Переписать список правил промпта так, чтобы он не содержал длинного списка аббревиатур в одной строке (они мигрируют в тело). Вынести список в отдельное system-turn сообщение или использовать более абстрактную формулировку («any uppercase Latin acronym»).
2. Пост-фильтр: если в переведённом сегменте появляется «LC, VNA, ICP, SNR, ADC, IEEE, GAI, RF» буквально — считать галлюцинацией и переводить сегмент повторно в single-mode.
**Критерий:** grep `«LC, VNA, ICP, SNR, ADC, IEEE»` по RU-HTML = 0.

### 🆕 Commit 4 — Sup/citation gluing fix (A2)
**Что:** Проверить и починить склейку `<sup>N</sup>` со следующим словом при переводе.  
**Где:** `translategemma.py` — сплит-логика по `_TAG_SPLIT_PATTERN`, и `single_file_html.py` — postfix-нормализатор пробелов вокруг `</sup>`.  
**Как:** после перевода добавить нормализацию: если `</sup>` плотно примыкает к `[A-Za-zА-Яа-яЁё]` — вставить пробел.  
**Критерий:** grep `</sup>[А-Яа-я]` = 0.

### 🆕 Commit 5 — Table titles and duplicates (A4)
**Что:** Проверить, что taблица не оставляется в двух экземплярах, а перевод заголовка связный.  
**Где:** `single_file_html.py` — поиск `<table>`; `translategemma.py` — специально для `<caption>` / `<thead>` использовать более аккуратный чанкинг.  
**Как:**
1. Убедиться, что оригинальная EN-таблица удаляется, а остаётся только RU.
2. Отдельно собирать текст caption/thead в более широкий контекст (полный заголовок + первая строка), чтобы «STATE OF ARTS» переводилось как «СОВРЕМЕННЫХ РЕШЕНИЙ», а не «ИСКУШЕНИЙ».
3. Доп-словарь терминов-заголовков в промпт/плейсхолдерах.
**Критерий:** отсутствие пары `TABLE`+`ТАБЛИЦА` в одном документе; ручная проверка перевода.

### 🆕 Commit 6 — Figure-caption anchor offset (B2)
**Что:** Клик на ссылку на рисунок должен показывать сам рисунок, а не подпись.  
**Где:** `single_file_html.py` — `_inject_default_styles` + `_add_figure_anchors`.  
**Как:**
1. Перенести `id="fig-N"` с `<p>` с подписью на `<figure>` или `<img>` (то, что стоит ВЫШЕ).
2. Альтернативно, добавить CSS `[id^="fig-"] { scroll-margin-top: -50vh; }` или аналогичный отступ, чтобы picture оказывалась в центре viewport.
**Критерий:** ручная проверка кликов по `<a href="#fig-N">` в браузере.

### 🆕 Commit 7 — Figures inside paragraphs (B1)
**Что:** Marker помещает `<figure>/<img>` внутрь `<p>`, разрывая предложение.  
**Где:** `single_file_html.py` — новый под-шаг в polish.  
**Как:** найти `<p ...>...<figure>...</figure>...</p>` и разрезать на `<p>...</p><figure>...</figure><p>...</p>` (склеивая порванные предложения по логике: если после фигуры текст начинается с строчной буквы — переносим его обратно к первой половине).
**Критерий:** grep `<p[^>]*>[^<]*<figure` и `<p[^>]*>[^<]*<img` в итоговом HTML = 0.

### 🆕 Commit 8 — URL+citation separator (C1)
**Что:** При переводе/поляришинге не терять пробелы между URL и следующим словом и восстанавливать `(ref. N)` как настоящую sup-ссылку.  
**Где:** `single_file_html.py` — `_autolink_plain_urls` + новый нормализатор `(ref. N)`, `(см. N)`, `(ref N)`.  
**Как:** добавить паттерн `\(ref\.?\s*(\d+)\)` → `<sup><a href="#ref-N">N</a></sup>` (с валидацией по ref_count).  
**Критерий:** grep `\(ref\.?\s*\d+\)` = 0.

### 🆕 Commit 9 — Bare-citation after space+dot (C2)
**Что:** Ловить паттерн `слово\s\d+\.` в конце предложения, когда после слова идят пробел + число + точка-конец-предложения, и число есть в ref-range.  
**Где:** `single_file_html.py` — новый `_BARE_CITATION_SENTENCE_END_PATTERN` + валидация по `ref_count`.  
**Паттерн:**
```
(?<=[A-Za-zА-Яа-яёЁ])\s(\d{1,3})(?=\.\s|\.$)
```
при условии `1 ≤ N ≤ ref_count`.  
**Критерий:** «клиническим наборам данных 62.», «проблемы 17.68.», «знаний 67.» — все стали кликабельными `#ref-N`. Ложные срабатывания на «5%», «версия 2.0» проверить регрессом.

### 🆕 Commit 10 — Caption backslash artifacts (D1)
**Что:** Расширить `_SLASH_PIPE_ARTIFACT_PATTERN` на одиночные `\` между словами в подписях.  
**Где:** `single_file_html.py:48` + полиш-шаги для caption.  
**Как:** `\s+\\\s+` → пробел (вне math-блоков `\[…\]`, `$…$`).  
**Критерий:** grep `\s\\\s` в подписях = 0.

### 🆕 Commit 11 — Title translation quality (A3)
**Что:** Улучшить перевод заголовка документа.  
**Где:** `translategemma.py` — специальный handling `<title>` / `<h1>` первого документа: более длинный контекст, явный prompt-hint «это заголовок научной статьи».  
**Критерий:** ручная проверка заголовка Wang.

## Универсальность (сквозные принципы)

1. **Валидация по реальным якорям библиографии.** Обёртки `<sup>N</sup>`/`[N]` → `<a href="#ref-N">` только если `id="ref-N"` существует в документе.
2. **Плейсхолдер-изоляция** непереводимого контента (аббревиатуры, URL, `<code>`, атрибуты).
3. **Конфигурируемые словари** — аббревиатуры, термины-заголовки выносятся из кода в `abbreviations.py`.
4. **Узкие триггеры** для math-распаковки и caption-чистки — не трогать реальные формулы/код.
5. Все фиксы универсальны: проверяются на обоих документах (Wang + Teo), не подгоняются под один PDF.

### Регресс-кейсы (тесты)
- «версия 2.0», «таблица 3.4», «GPT-4», «страница 17.68 содержит 5%» — не ссылки.
- Реальные формулы с буквами/операторами — не разворачиваются в sup.
- `<sup>a</sup>`, `<sup>b</sup>` (короткие маркеры таблиц) — не трогаются.

### Ручной чек-лист (Wang + Teo)
- Все Figure-картинки отображаются (и инлайнены в base64).
- Клик по `Рис. 2` показывает картинку, а не только подпись.
- `DeepSeek-R1`, `GPT-4` не кликабельны как ссылки.
- Отсутствуют рядом `AI` и `ИИ`.
- Заголовок статьи и таблиц — связный перевод.
- Нет фрагментов промпта в теле.
